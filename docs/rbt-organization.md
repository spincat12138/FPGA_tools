# RBT文件整理 子工具文档

本文档维护 `RBT_Organization` 子工具的业务规则、内部结构和验证要求。代码代理通用规则见根目录 `AGENT.md`。

## 工具定位

- 工具名称：RBT文件整理。
- 工具用途：扫描用户选择的目标目录，提取其中的 `.rbt` 文件并复制到该目录下的 `rbt/` 子目录；勾选 `同时导出 bit` 时，同步提取 `.bit` 文件到 `bit/` 子目录。
- 独立运行入口：`RBT_Organization/rbt_file_organization.py`。
- 主界面接入入口：`RBT_Organization/__init__.py` 暴露的 `create_widget(parent=None, services=None)`。

## 内部结构

- `core.py` 放目录校验、递归扫描、目标文件名计算和复制逻辑。
- `widget.py` 放 PyQt 界面逻辑、信号槽绑定和与 `ToolServices` 的交互。
- `rbt_file_organization.py` 保留命令行入口，并调用 `core.py` 中的同一份整理逻辑。

## 业务规则

- 输入必须是存在的文件夹路径。
- RBT 输出目录固定为输入目录下的 `rbt/`，扫描时会跳过该输出目录，避免重复整理已输出文件。
- 勾选 `同时导出 bit` 时，BIT 输出目录固定为输入目录下的 `bit/`，扫描时会同时跳过 `rbt/` 和 `bit/` 两个输出目录。
- `.rbt` 扩展名匹配不区分大小写。
- 子目录中的 `.rbt` 文件复制后，以输入目录下第一层子目录名作为目标文件名，例如 `case_a/foo/bar.rbt` 输出为 `rbt/case_a.rbt`。
- 直接位于输入目录下的 `.rbt` 文件保留自身主文件名。
- `.bit` 文件整理规则与 `.rbt` 完全一致，仅目标扩展名为 `.bit`，例如 `case_a/foo/bar.bit` 输出为 `bit/case_a.bit`。
- 若多个源文件会生成同一个目标文件名，或目标文件已存在，整理会失败并列出冲突来源，避免覆盖已有数据。

## GUI 维护约定

- 界面沿用项目现有 PyQt5 视觉基调，如有差异在此说明。
- 界面只提供整理路径输入、目录选择、开始整理、打开输出目录、日志和进度显示。
- 目录扫描和复制通过后台线程执行，避免阻塞主界面。
- 需要日志、弹窗、目录选择和忙碌状态时优先使用主界面传入的 `ToolServices`。

## 验证要求

- 修改核心整理规则时，至少验证：不存在路径、空目录、直接包含 `.rbt`、多层子目录包含 `.rbt`、输出目录跳过、冲突路径和已有目标文件场景。
- 修改 GUI 时，确认主界面中 `RBT文件整理` 标签页可以加载，选择目录后可以完成整理并打开输出目录。

## 完成标准

- 本工具可通过主界面标签页加载。
- 命令行入口仍可调用同一份核心整理逻辑。
- 输入校验、状态提示、进度显示和失败弹窗清晰。
